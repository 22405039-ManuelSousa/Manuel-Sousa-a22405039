Sobre esta Aplicação

Arquitetura MVT

Esta aplicação segue o padrão MVT (Model-View-Template) do Django O Model define a estrutura dos dados por exemplo, um Projeto tem título, descrição, imagem e tecnologias associadas. A View recebe o pedido do utilizador, consulta a base de dados através do Model, e decide o que mostrar por exemplo, projetos_view obtém todos os projetos e passa-os ao template. O Template é o ficheiro HTML que recebe esses dados e os apresenta ao utilizador. Quando alguém abre /projetos/, o Django encaminha o pedido pela URL para a view projetos_view, que consulta o modelo Projeto, e devolve a página projetos.html com a lista preenchida.

Modelação

Diagrama entidade-relação e descrição dos modelos A aplicação tem os seguintes modelos principais: Projeto, Tecnologia, Competencia, Formacao, UnidadeCurricular, Licenciatura, Docente, TFC, MakingOf, Conquista. Um Projeto pertence a uma UnidadeCurricular (ForeignKey) e pode usar várias Tecnologia (ManyToManyField). Uma Competencia relaciona-se com várias Tecnologia e vários Projeto. Uma Formacao associa-se a Tecnologia e Competencia. O modelo MakingOf documenta o processo de desenvolvimento e pode estar ligado a um Projeto, Tecnologia ou UnidadeCurricular.

Tecnologias Usadas

Django — framework Python usado para criar os modelos, views e URLs da aplicação. HTML/CSS — utilizados nos templates para estruturar e estilizar as páginas. SQLite — base de dados usada por omissão no Django para guardar todos os registos. Git/GitHub — controlo de versões do projeto e repositório remoto para guardar o código.
